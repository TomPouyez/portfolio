RootMe

Ce challenge consiste en une intrusion via une vulnérabilité d'upload de fichier et une élévation de privilèges Linux.

Le projet a débuté par une phase de reconnaissance avec NMAP, identifiant un serveur Apache sur le port 80. L'utilisation de GoBuster a révélé deux répertoires critiques : /panel/ pour l'upload de fichiers et /uploads/ pour leur exécution.

L'exploitation a consisté à contourner les restrictions d'upload en modifiant l'extension d'un reverse shell PHP (passage de .php à .phtml). Une fois le shell exécuté, j'ai obtenu un accès initial sur la machine, me permettant de récupérer le premier flag utilisateur.

Pour l'escalade de privilèges, j'ai recherché les fichiers possédant le bit SUID. L'exécutable Python a été identifié comme vulnérable. En utilisant une commande spécifique trouvée sur GTFOBins, j'ai pu contourner les restrictions locales pour obtenir un shell root et valider la machine.
